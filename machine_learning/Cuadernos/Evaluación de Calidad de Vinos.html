⭕️ Introducción¶

En la industria vin√≠cola, evaluar la calidad de un vino de manera precisa y consistente es crucial tanto para los productores como para los consumidores. Con una gran cantidad de caracter√≠sticas qu√≠micas y sensoriales, el an√°lisis individual de cada vino resulta complejo y poco pr√°ctico. Gracias al uso de modelos de machine learning y al an√°lisis de datos, es posible identificar patrones en las caracter√≠sticas qu√≠micas de los vinos que afectan su calidad. Este enfoque permite desarrollar modelos de clasificaci√≥n supervisada que predicen la calidad de un vino a partir de sus propiedades, optimizando el proceso de evaluaci√≥n y apoyando decisiones basadas en datos. Variables como la acidez fija, el contenido de az√∫car residual, la densidad y el nivel de alcohol, entre otras, se utilizan para crear modelos que puedan clasificar de forma confiable la calidad del producto, mejorando as√≠ la eficiencia y precisi√≥n en la industria.

⭕️ Núcleo Problémico¶

Dado el crecimiento en la producci√≥n y demanda de vinos de alta calidad, las bodegas enfrentan el reto de clasificar sus productos de manera eficiente y precisa entre las categor√≠as de vino "blanco" y "tinto". Sin una herramienta confiable para evaluar la calidad del vino a partir de sus propiedades qu√≠micas, la clasificaci√≥n y diferenciaci√≥n de productos puede volverse inconsistente y subjetiva, afectando la percepci√≥n del consumidor y la competitividad en el mercado. En particular, se necesita analizar si existe una relaci√≥n significativa entre caracter√≠sticas como la acidez, el contenido de az√∫car residual, el nivel de alcohol y otros componentes qu√≠micos, y la variable objetivo de clasificaci√≥n del vino ("class"), que distingue entre vinos "blancos" y "tintos". La ausencia de un modelo predictivo basado en datos dificulta la toma de decisiones informadas sobre el proceso de producci√≥n y calidad, limitando la capacidad de la bodega para optimizar sus productos y mejorar su posicionamiento en el mercado. Sin claridad en estas relaciones, la clasificaci√≥n de los vinos podr√≠a depender de criterios poco objetivos, lo cual afectar√≠a la reputaci√≥n de la marca y la satisfacci√≥n del cliente.

⭕️ Aplicación de Modelos de Machine Learning¶

⚛️ LogisticRegression¶

El modelo de Regresi√≥n Log√≠stica se utiliza para clasificar la variable objetivo quality_desc del conjunto de datos de vino, que representa la evaluaci√≥n cualitativa del vino. A continuaci√≥n, se describen los pasos y componentes clave de su implementaci√≥n:

Definición de Variables:¶
  • X: Contiene todas las caracter√≠sticas del DataFrame, excluyendo la columna quality_desc, que se utilizar√°n como variables predictivas para el modelo.
  • Y: Representa la variable objetivo, que es quality_desc, donde se clasifican las evaluaciones de calidad del vino.
División del Conjunto de Datos:¶

Se divide el conjunto de datos en conjuntos de entrenamiento y prueba utilizando train_test_split. Se asigna el 20% de los datos para pruebas (test_size=0.2), asegurando que el conjunto de entrenamiento sea suficientemente grande para un modelo robusto. La opci√≥n random_state=101 se utiliza para garantizar la reproducibilidad del muestreo aleatorio.

Escalamiento de Datos:¶

Se emplea un escala dor (StandardScaler) para normalizar las caracter√≠sticas de entrada. Este paso es crucial en modelos de regresi√≥n log√≠stica, ya que mejora la convergencia durante el proceso de optimizaci√≥n al ajustar los coeficientes del modelo.

Creación del Pipeline:¶

Se elabora un Pipeline que encadena dos tareas:

  • Escalado de las caracter√≠sticas mediante el StandardScaler.
  • Modelado utilizando LogisticRegression. Este enfoque simplifica el flujo de trabajo al permitir que ambos pasos se realicen de manera secuencial y coherente, facilitando la gesti√≥n del modelo.
